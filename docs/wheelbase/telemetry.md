# Telemetry

The Telemetry tab allows the wheelbase to generate additional Force Feedback based on **vehicle telemetry data** received from supported sim racing games.

Unlike standard DirectInput Force Feedback, Telemetry-based feedback is calculated by the VNM telemetry engine using real-time vehicle data.

## What Telemetry Can Reproduce

Depending on the selected FFB Mode, telemetry data can:

- Supplement the Force Feedback generated by the game, or
- Generate Force Feedback directly from vehicle telemetry data

Additional driving cues available via telemetry:

- Gear shifts
- Wheel slip
- Road surface detail
- ABS activity

---

## Telemetry Force Distribution

Telemetry Force Feedback consists of two components:

| Component | Description |
|---|---|
| **Telemetry Core Force** | Continuous forces from vehicle dynamics (steering load, etc.) |
| **Telemetry Effects** | Short-duration signals from specific events (gear shifts, ABS, wheel spin) |

### Force Allocation

**When no telemetry effects are enabled:**

- Telemetry Core Forces = 100%
- Telemetry Effects = 0%

**When one or more telemetry effects are enabled:**

By default, approximately **70%** is allocated to telemetry forces and **30%** to telemetry effects.

!!! tip
    Enable only the effects that provide meaningful feedback for your selected sim racing game. Too many effects simultaneously can cause noise in the continuous telemetry force.

---

## Telemetry Plugin

Telemetry data is provided through a plugin system.

### Plugin Types

**Native Plugins** — Distributed as DLL files, loaded directly by VNM SimCenter.

**Python Plugins** *(deprecated)* — Consist of `.vnp` and `.vne` files. These are deprecated and may be removed in future versions.

### Installing Plugins

**Step 1** — Open the **Plugin Store**.

**Step 2** — Select the desired plugin.

**Step 3** — Click **Install**.

**Step 4** — Click **Reload plugin** after installing to ensure it is properly loaded.

Alternatively, plugin files can be manually copied into the `plugins` directory located in the same folder as the VNM SimCenter application.

## Telemetry Effect Parameters

Each telemetry effect can be configured using these parameters:

| Parameter | Description |
|---|---|
| **Enable** | Enables or disables the effect |
| **Waveform** | Sine / Triangle / Square / Sawtooth / Constant |
| **Gain** | Overall amplitude of the generated force |
| **Frequency** | Oscillation frequency (higher = faster vibration) |
| **Duration** | How long the effect lasts after triggering (`-1` = no limit) |

---

## Telemetry Effects

### Gear Shift

Generates a short vibration or force pulse when the vehicle changes gear.

Especially useful in vehicles with sequential gearboxes or paddle shifters.

**Recommended Settings:**

- Use short duration values
- Use medium to high frequencies
- Avoid excessive gain that may mask other FFB signals

Typical duration: **10–40 ms**

---

### Engine Vibration (EngineVibe)

Generates continuous vibration based on engine speed (RPM).

As RPM increases, the vibration frequency increases to reflect faster engine operation.

**Recommended:**

- Use moderate gain values
- Use smooth waveforms (Sine)
- Avoid excessive vibration strength

**Example Baseline Configuration:**

| Parameter | Value |
|---|---|
| Waveform | Sine |
| Gain | 20 – 40 |
| Frequency | 40 – 80 Hz |

---

### Road Texture

Generates subtle vibrations based on road surface telemetry data.

Simulates small surface irregularities of the track.

Road texture is generated from real-time telemetry data — typically uses a **constant force signal**.

**Recommended:** Use moderate gain values and avoid excessive vibration strength.

---

### ABS

Generates vibration feedback when the vehicle's Anti-lock Braking System (ABS) is activated.

Helps detect when the braking system is approaching the traction limit.

**Parameters:**

| Parameter | Description |
|---|---|
| **Brake Input ABS Trigger** | Minimum brake pedal input required before ABS effect activates (prevents triggering during light braking) |
| **ABS Trigger** | Telemetry threshold to detect ABS activity (lower = more sensitive) |

!!! note
    Only iRacing uses the ABS flag to generate an ABS signal. Other games use different telemetry calculations.

---

### Brake Lock Up

Generates vibration feedback when wheels begin to lock during braking.

Especially useful in racing simulators without ABS or when ABS is disabled.

**Parameters:**

| Parameter | Description |
|---|---|
| **Brake Input BLU Trigger** | Minimum brake pedal input required before lockup feedback activates |
| **BLU Trigger** | Telemetry threshold to detect wheel lock (lower = more sensitive) |

**Recommended:** Square or Triangle waveforms for sharper, more perceptible pulses.

---

### Wheel Spin

Generates vibration feedback when driven wheels begin to spin faster than available tire grip during acceleration.

**Parameters:**

| Parameter | Description |
|---|---|
| **Throttle Trigger** | Minimum throttle input required before wheel spin effect activates |
| **Wheel Spin Trigger** | Telemetry threshold to detect wheel slip (lower = more sensitive) |

---

### Understeer

Generates vibration feedback when front tires begin to lose grip during cornering.

Typically uses lower to medium frequencies so the vibration remains noticeable but not distracting.

**Parameters:**

| Parameter | Description |
|---|---|
| **Understeer Trigger** | Telemetry threshold required before effect activates (lower = more sensitive) |

---

### Oversteer

Generates vibration feedback when rear tires begin to lose grip during cornering.

Square or Triangle waveforms are often preferred for clearer, more detectable pulses during rear traction loss.

**Parameters:**

| Parameter | Description |
|---|---|
| **Oversteer Trigger** | Telemetry threshold required before effect activates (lower = more sensitive) |

---

## General Telemetry Settings

### Haptic Only

When enabled, telemetry effects generate vibration feedback **without modifying the main steering torque**.

Use this to test each effect in isolation.

### Haptic Headroom

Reserves a portion of torque for telemetry effects.

| Value | Effect |
|---|---|
| Low | Stronger steering, weaker telemetry |
| High | Clearer telemetry, slightly reduced steering force |

**Default: 30%**

### Force Spike Reduction

| Parameter | Description |
|---|---|
| **Force Spike Reduction Enable** | Limits extremely large telemetry force spikes |
| **Force Spike Trigger** | Threshold at which spike reduction begins (lower = more sensitive) |
| **Force Spike Limit** | Maximum allowed force when spike reduction is active |

### Telemetry Condition Force

Applies condition effects at the telemetry processing stage. Useful for reducing oscillation caused by telemetry-generated forces.

| Parameter | Description |
|---|---|
| **Friction Gain** | Adds simulated friction to telemetry steering forces |
| **Damper Gain** | Adds damping to reduce oscillations |
| **Inertia Gain** | Adds simulated rotational inertia |

## Game Telemetry Activation

### iRacing

Modify the `app.ini` file to enable full telemetry support:

**Step 1** — Navigate to:
```
C:/Users/<your account>/Documents/iRacing
```

**Step 2** — Open `app.ini`

**Step 3** — Set the following parameters:
```ini
irsdkLog360Hz=1
enableFFB360HzInterpolated=1
```

### rFactor 2 / Le Mans Ultimate (LMU)

Install the shared memory plugin from:
[rF2SharedMemoryMapPlugin](https://github.com/TheIronWolfModding/rF2SharedMemoryMapPlugin/releases/tag/3.7.14.2)

**Step 1** — Download and extract the plugin.

**Step 2** — Copy to the game plugin folder:

- **rFactor 2:** `<rFactor2 installation folder>/Bin64/Plugins`
- **LMU:** `<LMU installation folder>/Plugins`

**Step 3** — Launch the game and ensure the plugin is loaded.

!!! warning
    Telemetry must be running for:

    - Telemetry FFB mode
    - Automatically Change Profile
    - Advanced effects (engine vibration, road effects, etc.)

    If telemetry is not detected: restart the game, restart VNM SimCenter, and check plugin installation path.
